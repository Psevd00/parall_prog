## 2 Теоретические сведения

### 2.1 MPI (Message Passing Interface)
MPI — стандарт для параллельного программирования с передачей сообщений, используемый в распределённых вычислительных системах. В отличие от OpenMP (общая память), MPI работает по модели **распределённой памяти**, где каждый процесс имеет свою собственную память, а обмен данными происходит через явную передачу сообщений.

### 2.2 Основные функции MPI
| Функция | Назначение |
|---------|------------|
| `MPI_Init` | Инициализация MPI |
| `MPI_Comm_rank` | Получение номера процесса |
| `MPI_Comm_size` | Получение количества процессов |
| `MPI_Bcast` | Рассылка данных всем процессам |
| `MPI_Scatterv` | Распределение данных между процессами |
| `MPI_Gatherv` | Сбор данных от всех процессов |
| `MPI_Wtime` | Точное измерение времени |
| `MPI_Finalize` | Завершение работы MPI |

### 2.3 Метрики производительности

**Ускорение (Speedup):**
\[S_p = \frac{T_1}{T_p}\]

**Эффективность (Efficiency):**
\[E_p = \frac{S_p}{p} = \frac{T_1}{p \cdot T_p}\]

**Производительность (GFLOPS):**
\[P = \frac{2N^3}{T \cdot 10^9}\]

## 3 Описание алгоритма

Программа читает две матрицы из файлов, выполняет их умножение с использованием MPI и сохраняет результат.

### 3.1 Распределение данных

При умножении матриц C = A × B используется следующий подход:
- Матрица A **разбивается по строкам** между процессами
- Матрица B **полностью дублируется** во всех процессах
- Каждый процесс вычисляет свою часть результирующей матрицы C
- Результаты **собираются на главном процессе**

#include <iostream>
#include <fstream>
#include <vector>
#include <mpi.h>

using namespace std;

vector<vector<double>> readMatrix(const string& filename, int& n) {
    ifstream file(filename);
    file >> n;
    vector<vector<double>> matrix(n, vector<double>(n));
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            file >> matrix[i][j];
    return matrix;
}

int main(int argc, char* argv[]) {
    MPI_Init(&argc, &argv);
    
    int rank, size;
    MPI_Comm_rank(MPI_COMM_WORLD, &rank);
    MPI_Comm_size(MPI_COMM_WORLD, &size);
    
    int n;
    vector<double> flatA, flatB, flatC;
    
    if (rank == 0) {
        // Чтение матриц
        auto A = readMatrix(argv[1], n);
        auto B = readMatrix(argv[2], n);
        
        // Преобразование в плоские массивы
        flatA.resize(n * n);
        flatB.resize(n * n);
        for (int i = 0; i < n; i++)
            for (int j = 0; j < n; j++) {
                flatA[i * n + j] = A[i][j];
                flatB[i * n + j] = B[i][j];
            }
        
        cout << "Matrix size: " << n << "x" << n << endl;
        cout << "Processes: " << size << endl;
    }
    
    // Рассылка размера
    MPI_Bcast(&n, 1, MPI_INT, 0, MPI_COMM_WORLD);
    
    // Рассылка матрицы B
    if (rank != 0) flatB.resize(n * n);
    MPI_Bcast(flatB.data(), n * n, MPI_DOUBLE, 0, MPI_COMM_WORLD);
    
    // Распределение строк A
    int rows_per_proc = n / size;
    int remainder = n % size;
    vector<int> sendcounts(size), displs(size);
    int offset = 0;
    for (int i = 0; i < size; i++) {
        sendcounts[i] = (i < remainder) ? (rows_per_proc + 1) : rows_per_proc;
        displs[i] = offset;
        offset += sendcounts[i];
    }
    
    vector<double> localA(sendcounts[rank] * n);
    MPI_Scatterv(flatA.data(), sendcounts.data(), displs.data(), MPI_DOUBLE,
                 localA.data(), sendcounts[rank] * n, MPI_DOUBLE, 0, MPI_COMM_WORLD);
    
    // Вычисление
    vector<double> localC(sendcounts[rank] * n, 0.0);
    for (int i = 0; i < sendcounts[rank]; i++)
        for (int j = 0; j < n; j++)
            for (int k = 0; k < n; k++)
                localC[i * n + j] += localA[i * n + k] * flatB[k * n + j];
    
    // Сбор результатов
    if (rank == 0) flatC.resize(n * n);
    vector<int> recvcounts(size);
    for (int i = 0; i < size; i++)
        recvcounts[i] = sendcounts[i] * n;
    
    MPI_Gatherv(localC.data(), sendcounts[rank] * n, MPI_DOUBLE,
                flatC.data(), recvcounts.data(), displs.data(), MPI_DOUBLE, 0, MPI_COMM_WORLD);
    
    if (rank == 0) {
        // Сохранение результата
        ofstream file("result_mpi.txt");
        file << n << endl;
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++)
                file << flatC[i * n + j] << " ";
            file << endl;

        cout << "Performance: " << ops / (elapsed * 1e9) << " GFLOPS" << endl;
    }
    
    MPI_Finalize();
    return 0;
}

### 8.2 Анализ ускорения

**Ключевые наблюдения:**

**Для малых матриц (N=200):**
- Ускорение на 4 процессах: 3.66x (эффективность 91.5%)

- Причина: накладные расходы на коммуникацию сравнимы с объёмом вычислений

**Для средних матриц (N=400-800):**
- Почти линейное ускорение до 4 процессов (эффективность 90-92%)
- На 8 процессах эффективность падает до 57-59%
- Оптимальное количество процессов — 4

**Для больших матриц (N=1200-1600):**
- Наилучшее масштабирование: ускорение 5.18x на 8 процессах для N=1200
- Эффективность на 8 процессах достигает 65%
- Вычислительная работа преобладает над коммуникацией

**Для максимальных матриц (N=2000):**

### 8.4 Закон Амдала

Оценка последовательной доли по данным для N=1200:

Ускорение на 8 процессах: **5.18x**

По формуле Амдала:

\[S_p = \frac{1}{f_s + \frac{1-f_s}{p}}\]

\[5.18 = \frac{1}{f_s + \frac{1-f_s}{8}}\]

\[f_s \approx 0.064\]

**Последовательная доля составляет около 6.4%**, что соответствует хорошей параллельной реализации.

Теоретическое максимальное ускорение:

\[S_{max} = \frac{1}{f_s} = \frac{1}{0.064} \approx 15.6x\]
