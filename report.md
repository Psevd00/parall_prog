## 2 Теоретические сведения

### 2.1 OpenMP
OpenMP (Open Multi-Processing) — API для параллельного программирования на языках C/C++ и Fortran. Позволяет распараллеливать циклы с помощью директив компилятора.

### 2.2 Основные директивы
- `#pragma omp parallel for` — распараллеливание цикла
- `collapse(2)` — объединение двух вложенных циклов
- `omp_set_num_threads()` — установка количества потоков
- `omp_get_wtime()` — точное измерение времени

### 2.3 Метрики параллельных программ

**Ускорение (Speedup):**
\[S_p = \frac{T_1}{T_p}\]

где:
- \(T_1\) — время выполнения на 1 потоке
- \(T_p\) — время выполнения на p потоках

**Эффективность (Efficiency):**
\[E_p = \frac{S_p}{p} = \frac{T_1}{p \cdot T_p}\]

## 4 Исходный код программы

```cpp
#include <iostream>
#include <vector>
#include <fstream>
#include <chrono>
#include <omp.h>

using namespace std;

vector<vector<double>> readMatrix(string filename, int& n) {
    ifstream file(filename);
    file >> n;

    vector<vector<double>> matrix(n, vector<double>(n));

    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            file >> matrix[i][j];

    file.close();
    return matrix;
}

void writeMatrix(string filename, vector<vector<double>> matrix) {
    ofstream file(filename);
    int n = matrix.size();

    file << n << endl;

    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++)
            file << matrix[i][j] << " ";
        file << endl;
    }

    file.close();
}

vector<vector<double>> multiply(vector<vector<double>> A, vector<vector<double>> B, int num_threads) {
    int n = A.size();
    vector<vector<double>> C(n, vector<double>(n, 0));

    omp_set_num_threads(num_threads);

    double start = omp_get_wtime();

    #pragma omp parallel for collapse(2) schedule(static)
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            double sum = 0.0;
            for (int k = 0; k < n; k++) {
                sum += A[i][k] * B[k][j];
            }
            C[i][j] = sum;
        }
    }

    double end = omp_get_wtime();
    cout << "OpenMP time: " << end - start << " seconds" << endl;

    return C;
}

int main(int argc, char* argv[]) {
    if (argc != 4) {
        cout << "Usage: " << argv[0] << " <matrix1_file> <matrix2_file> <num_threads>" << endl;
        return 1;
    }

    string fileA = argv[1];
    string fileB = argv[2];
    int num_threads = atoi(argv[3]);

    int n1, n2;

    vector<vector<double>> A = readMatrix(fileA, n1);
    vector<vector<double>> B = readMatrix(fileB, n2);

    if (n1 != n2) {
        cout << "Matrices must be same size!" << endl;
        return 1;
    }

    int n = n1;
    
    cout << "Matrix size: " << n << "x" << n << endl;
    cout << "Threads: " << num_threads << endl;

    vector<vector<double>> C = multiply(A, B, num_threads);

    writeMatrix("result.txt", C);

    long long operations = 2LL * n * n * n;
    cout << "Operations: " << operations << endl;

    return 0;
}
```

## 8 Анализ данных

Для малых матриц (N=200-400):
Оптимальное количество потоков — 8-12
Увеличение потоков свыше 8 даёт незначительный прирост из-за накладных расходов
Для N=200 лучшее время достигнуто на 8 потоках (0.0636 с), при 12 потоках время увеличивается до 0.0806 с

Для средних матриц (N=600-800):
Наблюдается хорошее масштабирование до 10 потоков
Для N=600 лучшее время на 10 потоках (1.0246 с)
Для N=800 лучшее время на 12 потоках (2.2125 с)

Для больших матриц (N=1200-2000):
Чётко видно улучшение производительности с ростом потоков до 12
N=1200: ускорение с 36.2 с (1 поток) до 5.7 с (12 потоков) — в 6.35 раза
N=2000: ускорение с 315.8 с до 87.9 с — в 3.59 раза
